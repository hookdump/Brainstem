# Maintainers

## Current maintainers

- `@hookdump` (project owner, release approver)

## Governance policy

- All changes merge through GitHub pull requests.
- At least one maintainer review is required before merge.
- CI must pass before merge.
- Security, auth, and data durability changes require explicit maintainer signoff.
- Breaking API changes must include migration notes in README and PR description.

## Maintainer responsibilities

- Triage and label issues.
- Review and merge pull requests.
- Keep dependency and release workflows healthy.
- Maintain roadmap and production hardening backlog.
